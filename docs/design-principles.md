# Enterprise API Framework Design Principles

1.
Tests never use Rest Assured directly.

2.
Clients expose business operations.

3.
Domain models represent API contracts.

4.
JSON parsing belongs to mappers.

5.
Factories create objects.

6.
Managers own lifecycle.

7.
Utilities contain no business logic.

8.
No reflection without measurable value.

9.
Architecture over convenience.

10.
Every abstraction must pay rent.
