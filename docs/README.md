## Documentation Principles

-   Read architecture before implementation.
-   Treat the Framework Reference as the source of truth.
-   Keep documentation synchronized with code changes.
-   Update CHANGELOG and versioning documents for every release.

------------------------------------------------------------------------

## Maintenance Checklist

Before every release verify:

-   Documentation matches implementation.
-   Links are valid.
-   Version numbers are correct.
-   Examples compile or execute.
-   Screenshots and diagrams (if any) are current.
